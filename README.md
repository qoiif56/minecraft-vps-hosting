## 为什么开我的世界服需要VPS？

先聊个很多人绕过去的问题：为什么不用自己电脑开服？

原因其实很现实。家用宽带大概率没有固定公网IP，就算用了内网穿透工具，稳定性也全看运气。电脑一关机，服务器就没了，朋友半夜想上线刷怪塔的计划当场落空。更别说电费、噪音这些隐性成本。

**VPS（虚拟专用服务器）** 解决的正是这些痛点：

和共享主机比，VPS 在性能和可控性上完全不是一个量级。MC 社区里也普遍建议"有预算就直接上VPS，别用面板服"，原因就是如此。

## 我的世界服务器VPS配置怎么选？

### 内存是最关键的指标

Minecraft Java版服务器对内存的需求比CPU更直接，基本上按人数和玩法来算：

| 玩家规模 | 推荐内存 | 建议CPU | 备注 |
|----------|----------|---------|------|
| 1–5人纯净/插件服 | 2 GB | 1核+ | 够用，轻量插件无压力 |
| 5–20人插件服 | 4 GB | 2核+ | 推荐起步配置 |
| 10–20人模组服（大型整合包） | 8 GB | 4核+ | 整合包内存消耗翻倍 |
| 50人以上 | 12 GB+ | 4核8核 | 建议分服或BungeeCord群组 |
| 百人服以上 | 16 GB+ | 8核+ | 需要专业运维 |

> **特别说明**：1.18以前版本的 Minecraft 服务端是单线程运行的，这意味着 **CPU 单核频率比核心数量更重要**。很多玩家踩过坑：花大价钱买了8核CPU，结果发现MC根本用不上那么多核，反而不如买个高频的2核机器跑得流畅。

### CPU频率为什么重要？

这是 Evoxt 这家商家的核心卖点所在。他们所有 VPS 的 CPU 时钟频率最高可达 **6.0 GHz**，远超同价位竞品普遍 2.2–2.4 GHz 的水平。对跑我的世界服务器来说，这个差距直接体现在：地图生成速度、复杂红石运作、大量实体同时运动时的 TPS（服务器每秒刻数）稳定性。

### 带宽和流量

玩家连接服务器主要消耗的是**上行带宽**，多人同时在线对流量的消耗不可小觑。一般来说：

- 10人以下小服：1Gbps端口 + 每月500GB流量已经绰绰有余
- 20–50人中型服：建议月流量在2000 GB以上

## Linux 上如何部署我的世界服务器？（基础教程）

拿到 VPS 之后，最简单的方式是用 Linux + Paper 服务端搭建。下面是基本流程，适合 Ubuntu 系统：

**第一步：连接服务器**

用 SSH 工具（如 Windows 的 PuTTY，或 Mac/Linux 直接终端）连接你的 VPS：

bash
ssh root@你的服务器IP


**第二步：安装 Java 运行环境**

Minecraft Java版服务端需要 Java 17 或更高版本：

bash
apt update && apt install -y openjdk-21-jre-headless


**第三步：下载服务端核心**

推荐使用 [Paper](https://papermc.io)（性能比原版服务端好很多）：

bash
mkdir minecraft && cd minecraft
wget https://api.papermc.io/v2/projects/paper/versions/1.21.1/builds/latest/downloads/paper-1.21.1-latest.jar -O server.jar


**第四步：启动并同意 EULA**

bash
java -Xms2G -Xmx2G -jar server.jar nogui
# 首次运行会生成 eula.txt，把里面的 false 改成 true
sed -i 's/false/true/' eula.txt


**第五步：后台常驻运行**

推荐用 `screen` 或 `tmux` 保持服务器在断开SSH后继续运行：

bash
screen -S minecraft
java -Xms2G -Xmx4G -jar server.jar nogui
# 按 Ctrl+A 再按 D 退出 screen，服务器继续后台运行


`-Xmx` 后面的数字是分配给 MC 服务器的最大内存，根据你的 VPS 内存调整（建议留出 512MB–1GB 给系统）。
